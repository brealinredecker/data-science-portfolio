<h1>Project Description</h1> <br>
This project focused on designing and building a Passenger Train Management System using a relational database approach. The goal was to address real-world challenges in train operations, such as inefficient ticket booking, lack of transparency around delays, and limited coordination between passengers, operators, and maintenance teams <br>I contributed to developing the full data architecture, starting with identifying data requirements and creating a conceptual model that defined key entities such as passengers, trips, trains, tickets, and stations, along with their relationships . This was followed by building a logical data model and normalized schema to reduce redundancy and ensure data consistency across the system. <br>

Beyond database design, I implemented core functionality using SQL, including stored procedures, functions, and triggers. These were used to handle complex workflows such as secure ticket purchasing, validating trip schedules, and automatically enforcing business rules (e.g., preventing bookings for past trips or canceling tickets when a trip is canceled). The use of transactions ensured data integrity through ACID principles, making the system reliable and robust. <br>

Overall, this project demonstrates my ability to design and implement a full database system from the ground up, combining data modeling, normalization, and advanced SQL techniques to solve complex, real-world problems. <br>
